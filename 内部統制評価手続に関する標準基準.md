# 内部統制評価手続に関する標準基準

## 1\. 手続の種類に関する基準（整備／運用）

内部統制評価の目的と時間的焦点を明確にするため、手続の種類を「整備状況の評価」と「運用状況の評価」に分類する。

### 1.1. 基準の定義

| 区分 | 定義 | 目的 | 時間的焦点 |
| :--- | :--- | :--- | :--- |
| **整備状況評価** | 識別されたリスクに対し、コントロールが①存在し、かつ②意図通りに機能すればリスクを有効に低減できるように設計されているかを評価する手続。 | コントロールの**設計上の有効性**を検証する。 | 特定の**一点**で評価する静的な評価。 |
| **運用状況評価** | 有効に設計されたコントロールが、評価対象期間を通じて、設計された通りに一貫して、かつ継続的に機能していたかを評価する手続。 | コントロールの**運用上の一貫性と継続性**を検証する。 | 一定の**期間**を対象とする動的な評価。 |

### 1.2. 手続のキーワードと具体例

評価計画やリスクコントロールマトリックス（RCM）を記述する際は、以下のキーワードと手続例を参考にする。

| 区分 | 主要手続 | キーワード（動詞・名詞） | 手続の具体例 |
| :--- | :--- | :--- | :--- |
| **整備状況評価** | **ウォークスルー** | ・追跡する<br>・質問する<br>・観察する<br>・閲覧する<br>・規程を確認する<br>・マニュアルを確認する | ・一件の取引を開始から終了まで追跡し、担当者に業務プロセスをヒアリングする 。<br>・業務規程や手順書を閲覧し、文書化されたルールと実務が一致しているか確認する 。<br>・担当者がシステムを操作する様子を観察する。 |
| **運用状況評価** | **サンプルテスト** | ・サンプリングする<br>・テストする<br>・突合する<br>・照合する<br>・再実施する<br>・継続的に<br>・一定期間 | ・評価対象期間の取引から25件を抽出し、承認記録と請求書を突合する 。<br>・月次で作成される調整表を複数月分入手し、作成と承認の証跡を確認する。<br>・システムログを閲覧し、特定の期間にアクセス権のない担当者による不正なアクセスがなかったことを確認する。 |

## 2\. 証跡の量に関する基準（大／中／小）

**1つの手続（サンプルテストの場合は1サンプル）あたり**に収集する証跡の範囲と深度を標準化するため、その量を「大」「中」「小」の3段階で定義する。

### 2.1. 基準の定義と具体例

| レベル | 基準 | 定義 | 手続の具体例 |
| :--- | :--- | :--- | :--- |
| **大 (High)** | **包括的証跡** | 1つの取引や事象に対し、その発生から完了までの一連の流れを証明するために、複数の異なる種類の証憑を網羅的に収集・検証する。 | ・非定型的なコンサルティング契約1件について、契約書、作業報告書、請求書、入金記録までの一連の証憑をすべて入手し、整合性を確認する。<br>・固定資産の除却1件に対し、除却申請の稟議書、現物確認の記録、会計処理の伝票、固定資産台帳の更新記録をすべて検証する。 |
| **中 (Medium)** | **標準的証跡** | 1つの取引や事象に対し、その正当性を証明する上で中核となる2～3種類の主要な証憑を収集し、相互に突合・照合する。 | ・購買取引1件に対し、発注書、納品書、請求書の3点を突合し、数量・金額・品名が一致することを確認する [1]。<br>・売上取引1件に対し、出荷指示書と請求書を照合し、売上計上基準を満たしていることを確認する。 |
| **小 (Low)** | **単一証跡** | 1つの取引や事象に対し、特定のコントロールが実施されたことを示す単一の証跡（承認印、記録の存在など）を確認する。 | ・経費精算申請書1件に対し、部長の承認印が押されていることのみを確認する。<br>・取締役会議事録1件を閲覧し、特定の議題が審議された記録が存在することを確認する [2]。 |

### 2.2. 基準の適用方法

この基準は、運用状況評価における**サンプルテスト**と組み合わせて使用する。評価者はまず、コントロールの実施頻度に応じてテスト対象とする**サンプル件数**を決定し、次に、その**各サンプル**に対してどのレベル（大／中／小）の証跡を収集するかを本基準に基づき決定する。

**適用例：**
「日次で行われる購買取引の承認」というコントロールを評価する場合。

1.  **サンプル件数の決定：** 日次取引のため、サンプル件数を「25件」と決定する 。
2.  **1サンプルあたりの証跡の量の決定：** 本基準に基づき、各取引の検証には「**中（標準的証跡）**」が適切であると判断する。
3.  **具体的な手続の策定：** 「抽出した25件の購買取引のそれぞれについて、発注書・納品書・請求書の3点を突合し、承認の記録を確認する」という評価手続を策定する。

## 3\. 手続の難易度に関する基準（難／中／易）

評価担当者のスキルセットと手続の要求レベルを適切に合致させ、監査品質を確保するため、手続の難易度を「難」「中」「易」の3段階で定義する。この分類は、手続を示す\*\*「動詞」が要求するスキルレベル\*\*を客観的な判断軸とする。

### 3.1. 基準の定義と要求スキル

| レベル | 基準 | 要求されるスキルと専門的判断 | キーワード（動詞） |
| :--- | :--- | :--- | :--- |
| **難 (Hard)** | **高度な専門的判断** | ・会計基準やITシステムに関する深い知識。<br>・複数の情報源を統合し、明確な基準がない状況で合理性を判断する高度な分析能力。<br>・複雑な計算や見積りの妥当性を評価する能力 。 | ・**評価する**<br>・**分析する**<br>・**妥当性を検討する**<br>・**再計算する**（複雑なもの）<br>・**検証する**（複雑なロジック） |
| **中 (Medium)** | **ルールに基づく照合** | ・細部への注意力と正確性。<br>・2つ以上の情報源を、定義されたルールや基準に照らして比較・照合する能力。<br>・社内規程や承認権限の理解。 | ・**照合する**<br>・**突合する**<br>・**整合性を確認する**<br>・**承認の有無を確認する** |
| **易 (Easy)** | **単純な確認・入手** | ・基本的なコミュニケーション能力と観察スキル。<br>・文書や記録の存在を確認したり、重要な相互検証を伴わずに情報を直接入手したりする能力。 | ・**閲覧する**<br>・**質問する**<br>・**観察する**<br>・**存在を確認する** |

### 3.2. 動詞に基づく手続の具体例

| レベル | 動詞 | 手続の具体例 |
| :--- | :--- | :--- |
| **難 (Hard)** | **評価する** | 貸倒引引当金について、債権のエイジング分析で用いられた仮定の合理性を評価する [3]。 |
| | **分析する** | 売上高と売上債権回転期間の相関関係を分析し、異常な変動がないか検討する（分析的実証手続） [4, 5]。 |
| **中 (Medium)** | **突合する** | 売上元帳に記録された取引を、請求書、出荷伝票、顧客注文書と突き合わせる [1, 6]。 |
| | **承認の有無を確認する** | 規定金額を超える購買取引について、購買申請書に担当部長の承認印があることを確認する。 |
| **易 (Easy)** | **閲覧する** | 取締役会議事録を閲覧し、重要な設備投資が審議・承認されていることを確認する [2]。 |
| | **質問する** | 在庫管理担当者に、棚卸の実施手続について口頭で質問する [7]。 |
